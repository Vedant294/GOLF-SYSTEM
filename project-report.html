📋 Project Overview

Golff is a full-stack subscription platform where golfers log their Stableford scores (1-45), which become lottery numbers in monthly prize draws. A portion of every subscription automatically funds a charity of the user's choice.

🗄️ Database Schema

Table Purpose
profiles User accounts, subscriptions, charity preferences
charities Charity partners with descriptions and images
scores User Stableford scores (max 5 per user, rolling)
draws Monthly draw records with winning numbers
draw_entries User participation in draws with match results
winners Prize winners with verification and payout status
donations Charity contributions (subscription + independent)
subscription_payments Payment history and invoice tracking
charity_events Upcoming charity events and golf days

💰 Prize Pool Distribution

Formula: Total Pool = (Active Subscribers × Subscription × 50%)
Distribution: 40% Jackpot (5-match) | 35% 4-match | 25% 3-match
Rollover: Jackpot carries forward if no 5-match winner

Example Calculation:

100 subscribers × ₹499 × 50% = ₹24,950 prize pool

  • Jackpot (40%): ₹9,980
  • 4-match (35%): ₹8,733 (split equally among winners)
  • 3-match (25%): ₹6,238 (split equally among winners)

🔐 Security Features

  • Row Level Security (RLS): Database-level access control on all tables
  • JWT Authentication: Secure session management with automatic refresh
  • Role Injection Prevention: Trigger hardcodes role='user' on signup
  • Stripe PCI Compliance: No card data stored locally
  • Storage Policies: Winner proofs restricted to owner + admin
