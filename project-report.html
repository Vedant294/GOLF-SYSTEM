📋 Executive Summary

Golff is a full-stack subscription platform that uniquely combines golf performance tracking, charitable giving, and gamified rewards through a monthly prize draw system. Built for the Digital Heroes Full-Stack Development Trainee Selection Process, this project demonstrates comprehensive full-stack development capabilities with modern technologies and best practices.

Core Concept: Golfers subscribe monthly (₹499) or yearly (₹4,999), log their Stableford scores (1-45), and those scores become their lottery numbers in a monthly prize draw. A portion of every subscription automatically funds a charity of the user's choice.

🔄 User Flows

1. New User Signup Flow

User visits /signup ↓ Fills account details (name, email, password) ↓ Selects plan (monthly/yearly) ↓ Chooses charity + contribution % (10-50%) ↓ Clicks "Pay Now" ↓ Supabase creates auth account ↓ Signs user in immediately ↓ Redirects to Stripe Checkout (hosted page) ↓ User enters test card: 4242 4242 4242 4242 ↓ Stripe processes payment (test mode) ↓ Stripe fires webhook to edge function ↓ Webhook activates subscription in DB ↓ Records payment in subscription_payments table ↓ Creates donation record for charity ↓ Sends welcome email via Resend ↓ Stripe redirects to /dashboard?success=true ↓ User sees active subscription + can add scores

2. Monthly Draw Flow

Admin goes to /admin/draws ↓ Configures month, year, draw mode (random/algorithmic) ↓ Clicks "Run Simulation" ↓ System generates 5 winning numbers (1-45) ↓ Fetches all active users' scores ↓ Calculates matches for each user ↓ Shows preview: X winners, prize breakdown ↓ Admin clicks "Publish Draw" ↓ Creates draw_entries for ALL active users ↓ Creates winners records for 3+ match users ↓ Calculates prize amounts (40/35/25% split) ↓ Rolls over jackpot if no 5-match winner ↓ Admin clicks "Broadcast Results" ↓ Sends personalized email to each subscriber ↓ Users see results on /draws and /dashboard

💰 Prize Pool Logic

Distribution Formula

Prize Pool Calculation:
Total Pool = (Active Subscribers × Subscription Amount × 50%)

Distribution:
• 40% → Jackpot (5-match) — rolls over if unclaimed
• 35% → 4-match winners — split equally
• 25% → 3-match winners — split equally

Example Calculation

100 active subscribers × ₹499 × 50% = ₹24,950 prize pool Jackpot: ₹9,980 (40%) 4-match: ₹8,733 (35%) 3-match: ₹6,238 (25%) If 2 users match 4 numbers: ₹8,733 ÷ 2 = ₹4,366 each If no 5-match: ₹9,980 rolls to next month's jackpot

Jackpot Rollover Mechanism

  • If no user matches all 5 numbers, the jackpot amount carries forward
  • Accumulated jackpot adds to next month's 40% allocation
  • Creates excitement and larger prizes over time
  • Tracked via jackpot_rolled_over boolean flag

🔐 Security Implementation

Authentication & Authorization

  • JWT-based Sessions: Supabase Auth with automatic token refresh
  • Password Validation: 8+ characters, uppercase, number required
  • Protected Routes: Client-side guards with auth state checks
  • Role-based Access: User vs Admin role enforcement

Database Security

  • Row Level Security (RLS): Enabled on all tables
  • Security Definer Functions: is_admin() to avoid recursive lookups
  • Role Injection Prevention: Trigger hardcodes role='user' on signup
  • Parameterized Queries: SQL injection prevention
  • Foreign Key Constraints: Referential integrity

Payment Security

  • Stripe PCI Compliance: No card data stored locally
  • Webhook Signature Verification: Validates Stripe events
  • Test Mode: No real charges during development
  • Service Role Key: Only used in edge functions, never exposed

Storage Security

  • Bucket Policies: Winner proofs restricted to owner + admin
  • File Upload Validation: Type and size restrictions
  • Secure URLs: Signed URLs for private content
